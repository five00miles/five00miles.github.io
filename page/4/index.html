发表于 2018-01-22 | 分类于 javascript

1. 执行环境以及作用域的概念

1.1 概念

执行环境（简称环境）定义了变量或函数有权访问的其他数据，决定了它们各自的行为。每一个执行环境都有一个与之关联的变量对象。环境中定义的所有变量和函数都保存在这个对象中。虽然我们编写的代码无法访问这个对象，但解析器在处理数据时会在后台使用它。

全局执行环境是最外围的一个执行环境。根据ECMAScript实现所在的宿主环境不同，表示执行环境的对象也不一样。在Web游览器中，全局执行环境被认为是window对象，因此所有全局变量和杉树都是作为window对象的属性和方法创建的。某个执行环境中的所有代码执行完毕后，该环境被销毁，保存在其中的所有变量和函数定义也随之销毁（全局执行环境直到应用程序退出——例如关闭网页或游览器——时才会被销毁）。

每个函数都有自己的执行环境。当执行流进入一个函数时，函数的环境就会被推入一个环境栈中。而在函数执行之后，栈将其环境弹出，把控制权返回给之前的执行环境。ECMAScript程序中的执行流正是在这个方便的机制控制着。

当代码在一个环境中执行时，会创建变量对象的一个作用域链。作用域链的用途：保证对执行环境有权访问的所有变量和函数的有序访问。作用域链的前端，始终都是当前执行的代码所在环境的变量对象。如果这个环境是函数，则将其活动对象作为变量对象。活动对象在最开始时只包含一个变量，即arguments对象（这个对象在全局环境中是不存在的）。作用域链中的下一个变量对象来自包含（外部）环境，而再一个变量对象则来自下一个包含环境。这样，一直延续到全局执行环境；全局执行环境的变量对象始终都是作用域链中的最后一个对象。

活动对象： 变量对象在真正执行时的另一种形式。未进入执行阶段之前，变量对象中的属性都不能访问！但是进入执行阶段之后，变量对象转变为了活动对象，里面的属性都能被访问了，然后开始进行执行阶段的操作。

变量对象：variable object—–VO

活动对象：Activation object—–AO

标识符解析是沿着作用域链一级一级地搜索标识符的过程。搜索过程始终从作用域链的前端开始，然后逐级地向后回溯，直到找到标识符位置（如果找不到标识符，通常会导致错误发生）。

js的数据类型

发表于 2018-01-15 | 分类于 javascript

js变量

  • js是弱类型的脚本语言
  • 变量的命名规则
    • 以字母或者下表符号_或者$为开头
    • 变量名长度不能超过255个字符
    • 不允许使用空格
    • 不使用脚本语言中保留的关键字和保留符号作为变量名
    • 变量名区分大小写
  • 全局变量
    • 在方法外部声明的变量
    • 方法内部，没有加var关键词声明的变量
  • 局部变量
    • 方法内部，使用var声明的变量

数据类型（6种）

1.基本数据类型

  1. undefined：值未定义
    • 变量声明了，但是没有被赋值
  2. null：空值，也是一个空对象
  3. boolen：布尔值

  4. number：数值

    • 整数，小数（最高精度17位小数），NaN，Infinity,-Infinity
    • var a = 070：表示八进制；var a = 0xA：表示十六进制
    • 不要做if(a+b==0.3)这样的判断，尽量避免小数判断

  5. string：字符串

2.引用类型

  1. Object：比如对象、数组、Date、正则表达式

与java没有任何关系，当时java很火，为了搭顺风车，就加个java

JavaScript和ECMAScript的关系：ECMAScript是一种由Ecma国际前身为欧洲计算机制造商协会，英文名称是European Computer Manufacturers Association,制定的标准。Javascript是由公司开发而成的，公司开发的一定又一些问题，比如不便于其他公司扩展和使用，所以欧洲的这个ECMA的组织，牵头制定JavaScript的标准，取名为ECMAScript。简单来说ECMAScript不是一门语言，而是一个标准。符合这个标准的常见语言有：JavaScript、Action Script(Flash)。
