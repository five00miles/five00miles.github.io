js函数表达式之执行环境及作用域

发表于 2018-01-22 | 分类于 javascript

1. 执行环境以及作用域的概念

1.1 概念

执行环境（简称环境）定义了变量或函数有权访问的其他数据，决定了它们各自的行为。每一个执行环境都有一个与之关联的变量对象。环境中定义的所有变量和函数都保存在这个对象中。虽然我们编写的代码无法访问这个对象，但解析器在处理数据时会在后台使用它。

全局执行环境是最外围的一个执行环境。根据ECMAScript实现所在的宿主环境不同，表示执行环境的对象也不一样。在Web游览器中，全局执行环境被认为是window对象，因此所有全局变量和杉树都是作为window对象的属性和方法创建的。某个执行环境中的所有代码执行完毕后，该环境被销毁，保存在其中的所有变量和函数定义也随之销毁（全局执行环境直到应用程序退出——例如关闭网页或游览器——时才会被销毁）。

每个函数都有自己的执行环境。当执行流进入一个函数时，函数的环境就会被推入一个环境栈中。而在函数执行之后，栈将其环境弹出，把控制权返回给之前的执行环境。ECMAScript程序中的执行流正是在这个方便的机制控制着。

当代码在一个环境中执行时，会创建变量对象的一个作用域链。作用域链的用途：保证对执行环境有权访问的所有变量和函数的有序访问。作用域链的前端，始终都是当前执行的代码所在环境的变量对象。如果这个环境是函数，则将其活动对象作为变量对象。活动对象在最开始时只包含一个变量，即arguments对象（这个对象在全局环境中是不存在的）。作用域链中的下一个变量对象来自包含（外部）环境，而再一个变量对象则来自下一个包含环境。这样，一直延续到全局执行环境；全局执行环境的变量对象始终都是作用域链中的最后一个对象。

活动对象： 变量对象在真正执行时的另一种形式。未进入执行阶段之前，变量对象中的属性都不能访问！但是进入执行阶段之后，变量对象转变为了活动对象，里面的属性都能被访问了，然后开始进行执行阶段的操作。

变量对象：variable object—–VO

活动对象：Activation object—–AO

标识符解析是沿着作用域链一级一级地搜索标识符的过程。搜索过程始终从作用域链的前端开始，然后逐级地向后回溯，直到找到标识符位置（如果找不到标识符，通常会导致错误发生）。


1.2 解释概念

1.2.1 变量对象(VO)的解释

某个执行环境下（假设是A），就只有上述代码块，那么这个A环境有一个与之关联的变量对象(VO)，假设是obj，那么这个obj对象里面保存了a,b,my()等变量和函数。

1.2.2 作用域链的解释

内部环境可以通过作用域链访问所有的外部环境，但外部环境不能访问内部环境中的任何变量和函数。这些环境之间的联系时线性、有次序的。每个环境都可以向上搜索作用域链，以查询变量和函数名；但任何环境都不能通过向下搜索作用域链而进入另一个执行环境。

如果想访问，可以使用闭包方法

2 词法分析

js代码是自上而下执行的，JavaScript并不是传统的块级作用域，而是函数作用域。JavaScript引擎会在代码执行前进行词法分析，所以事实上，js运行分为此法分析和执行两个阶段。

  • 第一步：先分析参数
  • 第二步：再分析变量声明
  • 第三步：再分析函数声明

一个函数能使用的局部变量，就从上面的3步分析而来。注意，是局部变量！！！

2.1 具体步骤：

  1. 函数运行前的一瞬间，生成活动对象(AO)
  2. 分析参数
    1. 把声明的参数，形成AO的属性，值全是undefined，例如AO.age = undefined
    2. 接收实参，形成AO相应属性的值,，即覆盖掉undefined的值
  3. 分析变量！！！声明！！！(加了var的那个)。如var age
    1. 如果AO上还没有age属性，则添加AO属性，值是undefined
    2. 如果AO上已经有age属性，则不做任何影响
  4. 分析函数声明，如function foo(){},则把函数赋给AO.foo属性，如果此前foo属性已经存在，则覆盖

3 延长作用域链

执行环境的类型只有两种——全局和局部（函数）

虽然执行环境的类型只有两种——全局和局部（函数）。但有些语句可以在作用域链的前端临时增加一个变量对象，该变量对象会在代码执行后被移除。在两种情况下会发生这种现象。具体来说，就是当执行流进入下列任何一个语句时，作用域链就会得到加长：

  • try-catch语句的catch块
    • 会创建一个新的变量对象，其中包含的时被抛出的错误对象的声明
  • width语句
    • 会将制定的对象添加到作用域链中

width语句接受的是location对象，因此其变量对象中包含了location对象的所有属性和方法，而这个变量对象被添加到了作用域链的前端。

4 没有块级作用域

js中没有块级作用域的概念，和c、java等高级语言不同。再c、java等高级语言中，if内定义的变量会在if语句执行完毕后被销毁。但在js中，if语句中的变量声明会将变量添加到当前的执行环境。所以在使用if、for时候要格外小心。

4.1 声明变量

使用var声明的变量会自动被添加到最接近的环境中。在函数内部，最接近的环境就是函数的局部环境；在width语句中，最接近的环境是函数环境。如果初始化变量时没有使用var声明，该变量会自动被添加到全局环境。

不要狭隘的直接把不带var的变量声明当作全局声明

4.1.1 声明变量加var和不加var

  • var i=100; -显式申明
  • i=100; -隐式申明

首先，它会尝试在当前作用域链中解析 num；
如果在任何沿着当前作用域链中往上找到num，则会执行对num属性赋值；

如果没有找到num，它才会在全局对象（即当前作用域链的最顶层对象，如window对象）中创造num属性并赋值,最后变成了window下的一个属性，window.num。

词法分析

词法分析阶段：

第一步：进行词法分析

  1. 形成AO={}
  2. 分析函数参数
    1. 没有形参。
    2. 没有实参。
  3. 分析var d,AO={d:undefined}
  4. 分析函数声明AO={t2:function(){d=5;e=6;}}

执行过程分析：

  1. t2();即执行function(){d=5;e=6;}
    1. d=5;在t2作用域内并没有声明d，向上查找在t1作用域内找到var d。则t2函数形成的AO={d:5}
    2. 执行到e = 6，在t2作用域中也找不到e,则向上寻找直至全局环境，都没找到。则在全局环境中声明e并把6赋值给e。即全局环境中的AO={e:6}
  2. console.log(e),全局作用域下AO={e:6}输出6
  3. console.log(d),全局作用域下并没有对d做定义，所以是d is not defined
